Detailed Description

This header contains various range-style functions that mimic std::algorithm functions. This won't be necessary anymore when ranges are introduced in C++20 (hopefully).

ossia::curve_segment

A curve segment is a function that maps a point between a start and an end.

Ex. :

double my_fun(double x, double start, double end)
{
return start + x * (end - start);
}

is the basic ease function that interpolates from start to end when x is between [0; 1].

In our case x can sometimes be greater than 1.

using ossia::destination_index = typedef boost::container::static_vector<char, 8>

Identifies an element in a nested tuple. For instance :

Tuple_A{
Tuple_B{1, 2, 3},
Tuple_C{
Tuple_D{4},
Tuple_E{5}
}
}
destination_index{} => Tuple_A
destination_index{0} => Tuple_B
destination_index{0, 1} => '1'
destination_index{0, 2} => '2'
destination_index{0, 3} => '3'
destination_index{1} => Tuple_C
destination_index{1, 0} => Tuple_D
destination_index{1, 0, 0} => '4'
destination_index{1, 1} => Tuple_E
destination_index{1, 1, 0} => '5'

There are at most 8 nested depth level.

ossia::expression

An expression is a formula that checks truth values according to values in a network tree.

For instance :

(a:/b == 50) && !(c:/d < e:/f)

Expressions are built with factories found in expression.hpp :

  • make_expression_true : a constant "true" node.
  • make_expression_false : a constant "false" node.
  • make_expression_atom : a relational comparison node (==, <=, !=, etc...) between two ossia::value.
  • make_expression_pulse : a node that becomes true if a message is received opn a given address since the last time update() has been called on it.
  • make_expression_not : standard logical not operator.
  • make_expression_composition : standard logical and, or, xor, operators between two expressions..
